# Quality and QA

## Strategy
- Unit tests for models and serializers
- API tests for permissions and workflows
- Integration tests for attendance and reports

## Acceptance Criteria (MVP)
- Key flows pass without manual fixes
- Permissions enforced for each role
- No critical security issues
- Attendance supports lesson-based time slots and academic-equivalent counts

## Environments
- Dev: local
- Staging: pre-production testing
- Production: dedicated client environment
